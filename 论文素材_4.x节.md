## 消融实验与模块贡献分析

为系统性地分析各创新模块对整体性能提升的贡献，本文设计了如下消融实验方案（`run_ablation_experiments.py`）：

### 实验配置

| 实验编号 | 配置 | 说明 |
|---------|------|------|
| **Model A (Full)** | Adaptive + Cascade + MI-RA-ToT | 完整系统 |
| **Model B** | Adaptive + Cascade, **w/o MI-RA-ToT** | 移除树式推理，回退为线性多跳 (M_core) |
| **Model C** | Adaptive + MI-RA-ToT, **w/o Cascade** | 移除级联路由，不进行置信度验证 |
| **Model D** | Cascade + MI-RA-ToT, **w/o Adaptive** | 移除自适应权重，使用固定等权检索 (端口 8001) |

各模块通过配置项独立启用/禁用：

```yaml
# config_v2.yaml - ablation 配置段
ablation:
  disable_adaptive_retrieval: false   # Model D 中设为 true
  disable_cascading: false            # Model C 中设为 true
  disable_tot: false                  # Model B 中设为 true
  force_strategy: null                # 可强制指定路由策略
```

### 模块贡献度量

通过比较完整系统 (Model A) 与移除单一模块后的系统性能差异，量化各模块的贡献：

```
ΔEM_MI-RA-ToT   = EM(Model A) - EM(Model B)    # 树式推理的贡献
ΔEM_Cascade     = EM(Model A) - EM(Model C)    # 级联路由的贡献
ΔEM_Adaptive    = EM(Model A) - EM(Model D)    # 自适应检索的贡献
```

ΔF1 的计算方式同上。通过这一消融实验设计，本文能够系统性地分析各模块对整体性能提升的贡献，同时评估各模块在不同数据集和问题类型上的表现差异。

### 二、自适应检索权重模型（创新点 1）

本模块通过分析查询的词法特异性与语义抽象度，动态计算 BM25、Dense、SPLADE 三路检索的融合权重。

**实现文件**：`retriever_server_v2/query_analyzer.py`

#### 特征计算

**词法特异性 (Lexical Specificity)**：衡量查询中实体信息的密度。

```
lexical_score = 0.7 × entity_ratio + 0.3 × length_bonus
```

其中：
- `entity_ratio = min(entity_token_count / total_tokens, 1.0)`
- `length_bonus = min(avg_entity_length / 3.0, 1.0)`
- 实体检测：使用 spaCy NER 模型 (`en_core_web_sm`)，并辅以启发式规则（大写序列、引号字符串、数字年份等）

**语义抽象度 (Semantic Abstractness)**：衡量查询的抽象/具象程度。

```
semantic_score = abstract_count / (abstract_count + concrete_count)
```

其中：
- 抽象关键词集合：`{how, why, relationship, significance, reason, effect, impact, influence, consequence, meaning, purpose, explain, describe, compare, contrast, analyze, evaluate, discuss, difference, similarity, connection}`
- 具象关键词集合：`{who, what, when, where, which, name, date, year, location, place, person, title}`

#### Softmax 门控机制

```python
alpha = 2.0       # 缩放因子
tau   = 1.0       # 温度参数 τ

logit_sparse = alpha × lexical_score      # BM25
logit_dense  = alpha × semantic_score     # Dense
logit_splade = (logit_sparse + logit_dense) / 2   # SPLADE (混合特性)

# 温度缩放 Softmax
logits = [logit_sparse, logit_splade, logit_dense]
scaled_logits = logits / τ
weights = softmax(scaled_logits)   # 归一化至 [0, 1]

# 输出：weights = {bm25: w1, splade: w2, dense: w3}
```

**温度参数 τ = 1.0**：当 τ → 0 时趋向 hard argmax（仅选择单路检索），当 τ → ∞ 时趋向均匀权重。τ = 1.0 为标准 Softmax，在区分度与平滑性之间取得平衡。

**示例**：

| 查询 | lexical | semantic | BM25 权重 | SPLADE 权重 | Dense 权重 |
|------|---------|----------|----------|------------|-----------|
| "Who directed Titanic?" | 0.18 | 0.20 | 0.327 | 0.333 | 0.340 |
| "Albert Einstein Nobel Prize year" | 0.85 | 0.10 | 0.423 | 0.321 | 0.256 |
| "How does climate change affect biodiversity?" | 0.05 | 0.90 | 0.215 | 0.322 | 0.463 |

### 三、级联动态路由机制（创新点 2）

当初次检索-生成的答案置信度不足时，自动级联至更强的推理策略（MI-RA-ToT）。

**实现文件**：`evaluate_v2/utils/confidence_verifier.py`

#### 置信度评估方法

采用交叉编码器 (Cross-Encoder) 进行后验置信度评估：

```python
def verify(question: str, answer: str, contexts: List[Dict]) -> float:
    # 1. 构造 QA 对
    qa_text = f"Question: {question}\nAnswer: {answer}"

    # 2. 使用 Cross-Encoder 对 QA 对与 Top-N 上下文进行相关性评分
    scores = []
    for ctx in contexts[:max_contexts]:     # 默认取 Top-3 上下文
        score = cross_encoder.predict([(qa_text, ctx['paragraph_text'])])
        scores.append(score)

    # 3. 取最大分数作为置信度（最佳支持性证据）
    confidence = max(scores)                # 归一化至 [0, 1]
    return confidence
```

#### 核心参数

| 参数 | 值 | 说明 |
|------|------|------|
| 置信度阈值 τ_conf | **0.6** | confidence < 0.6 时触发级联 |
| 验证模型 | cross-encoder/ms-marco-MiniLM-L-6-v2 | Cross-Encoder |
| 验证上下文数 | 3 | 取 Top-3 文档进行评分 |
| 级联目标 | M-ToT | 级联至树式推理策略 |
| 设备 | CUDA | GPU 加速推理 |

#### 级联决策逻辑

```python
def should_cascade(confidence: float, strategy: str) -> bool:
    # M 策略已是最强策略，不再级联
    if strategy.startswith("M"):
        return False
    # 置信度低于阈值则触发级联
    return confidence < threshold   # τ_conf = 0.6
```

即：对于 Z、S-Sparse、S-Dense、S-Hybrid 策略的初次回答，若 Cross-Encoder 评估置信度低于 0.6，则将该问题级联至 MI-RA-ToT 进行更深层次的多跳推理。M 策略本身已是最强推理路径，不再触发级联。

### 四、MI-RA-ToT 树式推理（创新点 3）

基于互信息引导的检索增强树式推理 (Mutual Information guided Retrieval-Augmented Tree-of-Thought)。

**实现文件**：`evaluate_v2/M_core_tot.py`

#### 核心参数

| 参数 | 符号 | 默认值 | 搜索范围 | 说明 |
|------|------|--------|---------|------|
| Beam 宽度 | B | **4** | 2-4 | 每层保留的路径数 |
| 最大深度 | max_depth | **4** | 3-5 | 树的最大搜索深度 |
| 候选查询数 | k | **3** | 2-3 | 每个节点生成的候选查询数 |
| 互信息相关性权重 | α | **0.65** | - | 文档-问题相关性权重 |
| 互信息新颖性权重 | β | **0.35** | - | 新信息增益权重 |
| 查询生成温度 | query_temperature | **0.3** | 0.2-0.4 | 较低温度产生更精确的查询 |
| 重排序模型 | - | cross-encoder/ms-marco-MiniLM-L-6-v2 | - | 用于相关性评估 |

以上超参数 (B, k, α, β) 均在验证集上通过网格搜索调优。

#### 互信息评分机制

MI 增益同时考虑文档的相关性与新颖性：

```
MI_gain = α × relevance(doc, question) + β × novelty(doc, existing_contexts)
```

**相关性计算 (Relevance)**：

```python
def calculate_relevance(question: str, doc: Dict) -> float:
    # 优先使用 Cross-Encoder 进行语义相关性评分
    score = cross_encoder.predict([(question, doc['paragraph_text'])])
    return clamp(score, 0.0, 1.0)
    # 回退：Jaccard 相似度
```

**新颖性计算 (Novelty)**：

```python
def calculate_novelty(new_doc: Dict, existing_contexts: List[Dict]) -> float:
    # 实体（标题）新颖性
    title_is_new = new_doc['title'] not in existing_titles
    title_novelty = 1.0 if title_is_new else 0.3

    # 内容新颖性（1 - 最大 Jaccard 相似度）
    content_novelty = 1.0 - max_jaccard_similarity(new_doc, existing_contexts)

    # 综合新颖性 = 0.4 × 实体新颖性 + 0.6 × 内容新颖性
    novelty = 0.4 × title_novelty + 0.6 × content_novelty
    return novelty
```

#### 树搜索流程

```
Step 0: 逻辑规划 (Logical Planning)
  └─ LLM 将问题分解为实体/概念推理链
     示例: "Tom Hanks → WWII movie → movie director → nationality"

Step 1: 初始检索
  └─ 对原始问题检索 8 篇文档，创建根节点 (score=0.0)

Step 2~4: Beam Search 迭代 (depth=1 to max_depth)
  For each node in beam (size B):
    ├─ (a) 生成 k 个候选查询 (temperature=0.3)
    ├─ (b) 执行检索 (每个候选 max_hits=5)
    ├─ (c) 计算 MI 增益: MI_gain = α×relevance + β×novelty
    ├─ (d) 创建子节点: score = parent_score + MI_gain
    └─ (e) 剪枝: 保留 Top-B 节点

Step 5: 选择最优路径
  └─ 选取最终 beam 中得分最高的节点，收集路径上的全部上下文

Step 6: 生成最终答案
  └─ 使用最优路径的所有上下文 + 逻辑规划 + 推理链，由 LLM 生成答案
```

#### 智能终止条件

```python
def _can_answer_question(node, question):
    if len(all_contexts) < 2:    return False    # 上下文不足
    if node.depth >= max_depth-1: return True     # 达到最大深度

    # 逻辑规划覆盖率检查
    if covered_plan_parts >= len(plan_parts) * 0.7:
        return True              # 70%+ 规划步骤已覆盖

    return len(all_contexts) >= 6  # 回退：至少 6 篇文档
```

## 4.x.4 评估指标 (Evaluation Metrics)

### 一、检索性能指标

| 指标 | 定义 | 说明 |
|------|------|------|
| **Recall@k** | 标准答案在 Top-K 检索结果中的命中率 | k = 5, 20, 100 |
| **Context Coverage** | 检索上下文对标准支持性文档的覆盖率 | 衡量检索的完整性 |
| **Avg Overlap** | 检索文档与标准文档的平均文本重叠度 | 衡量检索的精确性 |

### 二、问答性能指标

| 指标 | 定义 | 计算方式 |
|------|------|---------|
| **Exact Match (EM)** | 标准化后预测答案与标准答案完全一致 | 二值指标 (0/1)，取多个标准答案中的最大值 |
| **F1-score** | Token 级别精确率与召回率的调和平均 | 标准化后按 token 集合计算 Precision × Recall |
| **Accuracy** | 双向字符串包含匹配 | Gold ⊆ Pred 或 Pred ⊆ Gold 或全 token 覆盖 |

指标按以下维度聚合：
- **按数据集**：HotpotQA, Musique, 2Wiki, NQ, SQuAD, TriviaQA
- **按初始路由动作**：Z, S-Sparse, S-Dense, S-Hybrid, M
- **按是否级联**：Direct（直接回答） vs Cascaded（级联至 MI-RA-ToT）
- **Overall**：全部数据的加权平均

### 三、路由与推理效率指标

| 指标 | 定义 | 说明 |
|------|------|------|
| **平均检索次数** | 每个问题的实际检索调用次数 | Z=0, S=1, M=1~max_depth×k |
| **平均推理深度** | 树式推理的实际搜索深度 | 范围 [1, max_depth]，越小说明提前终止越高效 |
| **平均延迟** | 从接收问题到输出答案的端到端耗时 | 包含分类、检索、生成全流程 |
| **级联率 (Cascade Rate)** | 触发级联的问题占比 | cascaded_count / total_questions |

### 四、模块贡献分析指标

通过消融实验量化各创新模块的性能增益与计算开销变化：

#### 性能增益

| 指标 | 计算方式 | 含义 |
|------|---------|------|
| **ΔEM** | EM(Full) - EM(w/o Module) | 移除该模块后 EM 的下降量 |
| **ΔF1** | F1(Full) - F1(w/o Module) | 移除该模块后 F1 的下降量 |

具体计算：

| 创新模块 | 性能增益计算 |
|---------|-------------|
| 自适应检索权重 (Innovation 1) | ΔEM = EM(Model A) - EM(Model D) |
| 级联动态路由 (Innovation 2) | ΔEM = EM(Model A) - EM(Model C) |
| MI-RA-ToT 树式推理 (Innovation 3) | ΔEM = EM(Model A) - EM(Model B) |

#### 计算开销变化

| 指标 | 说明 |
|------|------|
| 平均检索次数变化 | 对比各实验中平均检索调用次数的增减 |
| 平均推理时间变化 | 对比各实验中端到端延迟的增减 |
| 级联触发率 | 仅 Model A 与 Model C 之间存在差异 |

通过上述指标体系，本文不仅能衡量各创新模块的绝对性能贡献，还能分析其性能-效率的权衡关系 (Performance-Efficiency Trade-off)，为实际部署提供参考依据。
